Gmail Integration — Setup Guide
📬
Connect Gmail to Operations Hub
This is a one-time 5-minute setup. Uses Google's official OAuth — Flood Roofing's emails never leave your browser.
⚠ Getting "Error 400: invalid_request"? You're missing Steps 4 or 5 below — the OAuth consent screen must be configured and your email added as a test user before it will work.
1
Go to Google Cloud Console
Visit console.cloud.google.com — sign in with office@floodroofing.co.nz
2
Create a new project
Top-left dropdown → "New Project" → name it Flood Roofing Hub → Create
3
Enable the Gmail API
Left menu → APIs & Services → Library → search Gmail API → click it → Enable
4
Configure OAuth Consent Screen ← this is the step most people miss
Left menu → APIs & Services → OAuth consent screen
• User Type: External → Create
• App name: Flood Roofing Hub
• User support email: office@floodroofing.co.nz
• Developer contact email: office@floodroofing.co.nz
→ Save and Continue
• Scopes page: click Add or Remove Scopes → search gmail.readonly → tick it → Update → Save and Continue
• Test users page: click + Add Users → enter office@floodroofing.co.nz → Add → Save and Continue
5
Create OAuth credentials
Left menu → APIs & Services → Credentials → + Create Credentials → OAuth Client ID
• Application type: Web application
• Name: Flood Roofing Hub
6
Add authorised JavaScript origin
Under "Authorised JavaScript origins" click + Add URI and enter exactly:
http://localhost:8099
⚠ Do NOT use file:// — Google rejects it. Leave "Redirect URIs" blank. → Create
7
Open this hub from the local server
Open Command Prompt in the folder where this file is saved and run:
python -m http.server 8099
Then open: http://localhost:8099/FloodRoofing_Hub_v6.html
8
Paste your Client ID below and connect
Back in Google Console → your new credential → copy the Client ID (ends in .apps.googleusercontent.com) → paste it here
